# FinProof Subagent Orchestration Flow Design

## Visual Philosophy: Evidence Grid

Evidence Grid treats an AI compliance pipeline as a calibrated decision instrument. The composition uses a restrained grid, thin connector lines, and clearly weighted nodes so that cost, judgment, and evidence boundaries are visible without long explanation.

The visual language separates always-on domain judgment from conditional quality gates. Default agents sit in the central rail as the primary inspection surface, while conditional agents branch below as guarded calls that only activate when evidence, case history, or conflict conditions justify extra model cost.

Color is functional rather than decorative. Product evidence, policy knowledge, domain analysis, conditional verification, and persistence each receive distinct but muted tones. The final piece should feel meticulously crafted, with every alignment, label, and spacing decision deliberate.

Typography is sparse and diagrammatic. Korean labels name the operational role; short English task IDs anchor the implementation contract. The diagram should read as a master-level systems artifact, not as a marketing graphic.
